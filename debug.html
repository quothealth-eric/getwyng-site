Step 1: Test the Demo

  1. Go to demo.html
  2. Upload your test files
  3. Open browser console (F12 ‚Üí Console tab)
  4. Look for detailed debug output starting with === API RESPONSE DEBUG ===

Step 2: Check Vercel Function Logs

  1. Go to Vercel Dashboard
  2. Click on your getwyng-site project
  3. Click the "Functions" tab at the top
  4. Click on /api/analyze function
  5. Click "View Function Logs"
  6. Test the demo, then refresh the logs to see new entries

Step 3: Alternative - Check Deployment Logs

  1. In Vercel Dashboard ‚Üí your project
  2. Click the "Deployments" tab
  3. Click on the most recent deployment
  4. Click "View Function Logs" or "Runtime Logs"

What to Look For

In Browser Console:

  • Full results: {...} - Complete API response
  • Line audit codes: [...] - Extracted CPT codes
  • Any error messages

In Vercel Logs:

  • PDF file read: X bytes - File upload success
  • PDF text extracted: X characters - OCR success
  • GPT extracted codes: [...] - What GPT found
  • MOCK DATA DETECTED or SUSPICIOUS warnings
